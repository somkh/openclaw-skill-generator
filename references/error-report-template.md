# 错误报告模板

在 Phase 5 的测试过程中遇到错误时，使用以下格式向用户报告。

---

## 标准错误报告格式

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔴 执行错误报告
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📍 错误位置：  [出错的文件名 + 行号或步骤编号]
               例1：scripts/fetch_data.js 第 47 行
               例2：SKILL.md → 执行步骤 → Step 3（调用外部 API）

💬 错误原因：  [用非技术语言解释为什么出错，避免使用只有程序员才懂的词]
               例1：脚本尝试读取文件 config.json，但该文件不存在于预期位置。
               例2：向天气 API 发送请求时返回了 401 错误，表示 API 密钥无效或未配置。

🛠  修改计划：
   1. [第一步，写到可以直接操作的程度]
      例：在 SKILL.md 的执行步骤 Step 1 中，增加对 config.json 是否存在的检查，
         如果不存在则返回明确的错误提示而非崩溃。
   2. [第二步（如有）]
   3. [第三步（如有）]

⚠️  影响范围：  [此错误是否影响 skill 的其他部分？]
               例：此错误只影响当前步骤，其他步骤不受影响。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

## 错误报告后的标准问句

输出错误报告后，始终接着问：

> 以上是我的修改计划，你是否同意？如果有不同意见请告诉我，我会调整后再执行。

---

## 常见错误类型及解释方式

| 错误类型 | 对用户的解释方式（非技术语言） |
|---------|-------------------------------|
| 文件不存在 | "脚本尝试读取 X 文件，但该文件不存在" |
| 权限拒绝 | "没有权限在 X 目录中写入文件" |
| 命令未找到 | "需要 X 工具但系统中未安装" |
| JSON 解析失败 | "收到的数据格式不是预期的 JSON 格式" |
| 网络错误 | "无法连接到 X 服务，可能是网络问题或服务暂时不可用" |
| 变量未定义 | "脚本在某处用到了一个没有值的变量，通常是参数没有正确传入" |
| 类型错误 | "传入的参数类型不对，例如预期是数字但收到了文字" |
